# Contributing to RTLDI Atlas

Thank you for your interest in contributing to the RTLDI Atlas project!

## How to Contribute

### Reporting Issues
- Use the GitHub Issues tab to report bugs, suggest improvements, or request new features.
- Please include as much detail as possible (data version, steps to reproduce, expected vs actual behavior).

### Suggesting Changes
- Fork the repository and create a feature branch from `main`.
- Make your changes.
- Submit a Pull Request with a clear description of what was changed and why.

### Running the Pipeline Locally
1. Clone the repository
2. Install dependencies (see README.md)
3. Download the required V-Dem and World Bank data
4. Run the canonical pipeline documented in README.md:
   - `python -m src.build_atlas --year 2026 --eta 0.30`
   - `python -m src.generate_enclosure_map --year 2026`
   - `python -m src.generate_atlas_ebook`
   - `python -m src.generate_html_atlas`

## Canonical Data Pipeline

The repository has one canonical data pipeline: `src.build_atlas` produces the atlas table, `src.generate_atlas_ebook.prepare_atlas_data()` prepares the shared print/web data, and the PDF/HTML generators consume that same data.

Do not add a separate JSON, spreadsheet, or frontend-only data pipeline that recomputes R, GDP baselines, country coverage, regional totals, or capital exclusions independently. Any web UI, fork, or visualization must consume outputs generated by the canonical pipeline or clearly document itself as an external experiment outside the official atlas.

Generated HTML must include all 193 UN Member States from the canonical 2026 breakdown unless the README explicitly documents a narrower edition.

### Code Style
- Keep changes focused and well-documented.
- Follow the existing structure in `src/` and `docs/`.

## Questions?
Open an issue or reach out via the project maintainer.

We appreciate contributions that help make this resource more accurate and useful for researchers and policymakers.
